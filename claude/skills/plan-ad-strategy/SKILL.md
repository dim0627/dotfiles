# plan-ad-strategy（広告戦略立案）

X(Twitter)広告クリエイティブの**戦略**を立てる。「どのインサイトに、どの訴求軸で、どんなメッセージを、どの器（表現モード）で当て、各スロットのコピーをどう振り、何を1変数として検証するか」を**複数候補**で出力する。

このスキルは**良い戦略候補を複数出すまで**が仕事。1つに絞らない（絞り込みは人の握り）。出力は**それ自体で完結する戦略まとめ**で、誰が受け取って制作してもよい（人間のデザイナー、画像生成、カンプ作成、いずれも一級の消費者）。**実際にどう制作するかには踏み込まない**。

## アウトプットの流れ

```
docs/product.md（＋過去出稿）
      │
      ▼
[plan-ad-strategy] ── 戦略候補を複数出す（Markdown）
      │
      ▼
  人が握る ── どの戦略を今回撃つか選ぶ（自動確定しない）
      │
      ▼
   制作 ── 選ばれた戦略が要求するクリエイティブを作る
```

## 管轄マップ（どこまでがこのスキルか）

| レイヤー | 中身 | 管轄 |
|---|---|---|
| **戦略** | インサイト / 訴求軸 / メッセージ方向 / **器の選択（表現モード）** / **全スロットのコピー方向** / 検証仮説 | **このスキル** |
| **制作** | 比率・背景の絵柄・配色・フォントサイズ・実コピーの文字組み・背景素材の生成・PNG 書き出し | 下流（制作） |
| **配信** | 入札・ターゲティング・予算・最新の広告メニュー確認 | 配信 / 人 |

> 「その戦略に必要なクリエイティブ」を2つに割る。**クリエイティブ・ブリーフ（＝方向：器の選択＋全スロットのコピー方向＋理由）はこのスキルの出力に入る**。**クリエイティブ実物（＝実コピーの文字組み・絵柄・比率・px）は下流の制作**。この線引きを越えない。

## 前提（満たさなければ止まる）

- **対象プロダクトの `docs/product.md` が存在すること**。コンセプト/ペルソナ/ポジショニング/トーン&ボイス/ブランド資産をここから読む。**無ければ「先に docs/product.md を整備してください」と伝えて停止する**（ペルソナや訴求を憶測で捏造しない）。
- ペルソナが `docs/product.md` で `[要確認]` 等の未確定なら、戦略の精度が落ちる旨を明示し、確定をユーザーに促してから進む。

## 入出力

- **入力**:
  - 対象プロダクトの `docs/product.md`（コンセプト/ペルソナ/コア体験/ポジショニング/トーン&ボイス/ブランド資産）
  - 過去出稿の実績（Notion 広告レジストリ。出稿0回ならスキップ可）
  - プロのマーケ/デザイン原則（このスキルに蒸留済み・下記「思考フレーム」）
- **出力**: 次に撃つクリエイティブの**戦略候補（複数）**を、人が読んで選べる **Markdown の戦略まとめ**で提示する（下記「出力フォーマット」）。具体コピーは plain text の「方向」まで。制作スペック（比率・絵柄・px）は含めない。

## X広告のアナトミー（コピーを振る先のスロット）

戦略は「全スロットのコピー方向」を出す。X広告1枚は次のスロットを持つ（最新メニューは配信時に確認）:

- **投稿本文**: ツイート文（メディアの上）。
- **メディア（画像/動画/カルーセル）内コピー**: 画像に乗る文字（小ラベル / 見出し / サブ）。カルーセルは**2〜6枚**スワイプで、各カードに個別のコピー＋遷移先を持てる。
- **カード見出し**: Website Card の見出し（**目安70字以内**で安定表示）。
- **CTA ボタン**: 「詳しくはこちら」等のアクション。

> セーフゾーン注意: 画像の**左下**に X 側のキャプション/カードUIが重なる。重要要素を左下隅に置かないことは戦略段階で意識する（具体の配置は制作の管轄）。

## 進め方（Who → What → How の順序原則）

**これは番号を機械的に踏むパイプラインではなく、上流から下流へ決める順序原則**。表現の器から入らず、Who（誰の何のジョブか）→ What（何を訴求し何と言うか）→ How（どの器で、どのスロットに、どう検証するか）の順で固める。各レベルに下記の検証済みフレームを当てる。

### Who — インサイトを掘る
ペルソナの**インサイト**（まだ言語化されていない本音）をニーズの3層で掘る:

1. **ニーズ**（自覚された必要性）
2. **潜在ニーズ**（未言語化の欲求・不満）
3. **インサイト**（既成概念に隠れた、行動の本当の動機）

掘る道具に **JTBD（ジョブ理論）**:
- ジョブを **状況(Context) × 動機(Motivation) × 成果(Outcome)** で書く。
- ジョブステートメント: **「[状況]のとき、[動機]したい。なぜなら[成果]を得たいから」**。
- 乗り換えは **進歩の4つの力**で起きる: **Push(現状の不満) + Pull(新解決策の魅力) > Anxiety(変化への不安) + Habit(惰性)**。優れた訴求は Pull を増やすだけでなく **Anxiety を下げる**点も狙う。

### What — 訴求軸を複数出し、メッセージに変換する
訴求軸 = **広告で押す戦略的な切り口**（≠ 具体コピーそのもの）。1つのニーズから**複数の訴求軸**を体系的に出す:

- **機能 / 感情 / 社会の3層**で軸を出す（機能だけに偏ると感情・社会層を取りこぼす）。例: 睡眠アプリ → 快適な睡眠(機能) / 朝スッキリ前向き(感情) / 自己管理できている自分(社会)。
- 各軸は **ベネフィット訴求**で書く: 機能そのものでなく「その機能が**何を解決するか**」。機能列挙（フィーチャーセリング）にしない。
- 発散に詰まったら **消費者ベネフィット6質問**: ①どんな気持ちになりたいか ②誰に認められたいか ③どんな人物でありたいか ④何に楽しみ/不安を感じるか ⑤どんな快楽/苦痛があるか ⑥この機能は競合とどう差別化されるか。

各訴求軸を主役メッセージに変換する。順序は **ニーズ → 価値 → 訴求軸 → コピー**:

- 機能への「直接的な答え」でなく「**どういう価値で応えるか**」を言語化する。
- **「買うときの気持ち」を起点**に現状へさかのぼって設計する（機能属性からでなく、感情・社会的承認・自己同一性の動機から組む）。
- 製品価値は**活用の文脈（状況・目的）**で決まる。機能列挙でなく利用文脈に紐づける。

> 3層はあくまで**発散の思考フレーム**であり測定可能な法則ではない。軸を広く出す道具として使い、絞り込みは下記「判断ポイント」で人に返す。

### How — 器を選び、全スロットにコピー方向を振り、検証仮説を立てる

**器（表現モード）はメッセージ構造から落ちてくる**。独立した工程として器ありきで選ばない。メッセージが「どう読まれる必要があるか」で器が決まる:

- **単発静止画**: 一撃で刺す単一メッセージ。段階的な見せ方が不要。
- **カルーセル（2〜6枚）**: ビフォー→アフター、段階的な説明、複数便益の列挙が肝。**何枚で何を見せるか**まで方向づける。
- **動画**: 時間軸・動き・実演が便益の核。
- **テキスト主体**: 会話的・口コミ的に読ませたい。

器は**名前まで決める**（コピー構造が器に依存するため。例: カルーセルなら何パネル分のコピーが要るかが変わる）。**理由を必ず添える**（なぜこの器か、をメッセージ起点で1文）。**ただし制作スペック（比率・絵柄・配色・px）は出さない**——そこは下流の制作の管轄。

器が決まったら、**全スロットにコピーの方向**を振る（投稿本文 / メディア内コピー / カード見出し / CTA）。各スロットは plain text の「方向」で書き、文字組み・装飾・サイズは制作に委ねる。

最後に**検証仮説（1変数）**を立てる。**1クリエイティブ = 1訴求軸 / A/B は1変数限定**（複数同時変更は因果が同定できない）。検証対象は**大要素 → 小要素**で、出稿ステージに応じて変える:

1. **初期**（勝ち筋未確定）: 大要素 = **訴求軸 / ビジュアル素材 / トンマナ**を検証。
2. **中期**（勝ち軸が見えた）: 確立した訴求軸の中で**コピーを精緻化**。
3. **成熟期**: **デザイン / 表現モード**を最適化し、クリエイティブ疲弊を防ぎリーチを広げる。

各仮説は「**何を変えて、何が上がると見るか＋なぜ**」を1文で書く。

## 出力フォーマット（Markdown・複数候補）

戦略候補を**複数**、人が比較して選べる体裁で並べる。各候補は次の構造:

```markdown
## キャンペーンゴール
サイト流入 / お試し登録（product.md・ユーザー確認）

## ペルソナ・インサイト
記録は続けたいが、食べたものを責められる感じが嫌で離脱する

---

## 候補A：罪悪感ゼロ
- **訴求軸**: 罪悪感ゼロ（層: 感情）
- **JTBD**: ダイエットに疲れたとき、責められずに記録だけ続けたい。前向きでいたいから
- **ベネフィット**: 記録しても責められない＝続けられる
- **メッセージ方向**: ダイエットの説教はしない。ただ記録するだけ
- **表現モード**: 単発静止画 ／ 理由: 一撃で刺す単一メッセージで段階的な見せ方は不要だから
- **スロット別コピー方向**:
  - 投稿本文: 続かなかったのは意志のせいじゃない、という共感から入る
  - メディア内: 「罪悪感は、いらない。」を主役に、説教しないトーン
  - カード見出し: 責めない記録、という約束を端的に（目安70字以内）
  - CTA: 無料で軽く始められることを示す
- **検証仮説**: 感情軸(罪悪感ゼロ)は機能軸(入力のゆるさ)よりCTRが高い。主ペルソナの離脱要因が罪悪感だから
- **今回動かす変数**: 訴求軸

---

## 候補B：入力のゆるさ
（同じ構造で別軸。層: 機能 など）

---

## 候補C：…
（社会層など、機能/感情/社会を取りこぼさない）

## 判断ポイント（あなたが握る）
- どの候補を今回撃つか（機能/感情/社会のどの層から攻めるか）
- …（下記「判断ポイント」参照）
```

- 候補は**機能/感情/社会の3層を取りこぼさない**ように出す（最低でも層をまたいで複数）。
- カルーセルを選んだ候補は、メディア内コピーを**パネル順の方向**で書く。
- コピーはすべて plain text の「方向」。**比率・絵柄・px は持たせない**。

## mandatories（戦略段階で必ず効かせる）

- **1クリエイティブ = 1訴求軸**（混ぜない）。A/B は1変数限定。
- **競合貶め表現 NG**（例「あすけんに挫折した人へ」）。競合は補完関係として扱う。
- **AI 生成写真は避ける**（日本市場は AI 生成感への拒否反応が強い）。イラスト/グラフィック系を選ぶ（最終判断は product.md のトーンに従う）。
- 表現の器から入らない。**Who → What → How** の順を守る。
- **制作スペック（比率・絵柄・配色・px）を出力に漏らさない**。出すのは器の名前と各スロットのコピー方向まで。

## 判断ポイント（人に握りを返す）

このスキルは戦略を**自動確定しない**。以下は機械的に決めず、候補を複数出してユーザーに選んでもらう。知見調査でも型が確立していない空白でもある:

- **どの候補を今回撃つか**＝3層（機能/感情/社会）のどの軸から攻めるかの最終選択（1クリエイティブ=1軸に絞る判断）。
- **低トラフィック環境で統計的有意に達しないとき**の暫定判断（最低出稿額・必要配信量・有意未満での打ち切り基準）。
- **Anxiety/Habit を下げる具体表現**（社会的証明・返金保証・移行コスト訴求等）の採否。

## 思考フレーム リファレンス（蒸留元・確度の明示）

deep-research（2026-06-09・敵対的検証済み）でプロの戦略立案知見を調査し、**化石化しにくい普遍プロセスのみ**採用した。下記は**高確度の検証済みフレーム**であり、上の「判断ポイント」は**型が未確立の空白**——確度を取り違えない。

- **訴求軸4段階フロー**: 3C分析 → ニーズ言語化 → （統計があれば）優先度付け → 自社の独自の強みと掛け合わせ。順序は Who → What → How。
- **JTBD（ジョブ理論）**: ジョブ = 状況×動機×成果。機能/感情/社会の3層。進歩の4つの力（Push+Pull > Anxiety+Habit）。出典系統: Christensen / Ulwick(Strategyn) / Moesta。
- **ベネフィット訴求**: 機能でなく便益（Features tell, benefits sell）。消費者ベネフィット6質問（FICC）。
- **インサイト3層**: ニーズ → 潜在ニーズ → インサイト。
- **A/Bテスト設計**: 1変数原則 / 大要素→小要素 / ステージ別検証。

> 検証で**反証され不採用**にしたもの: 「訴求軸の8パターン固定分類」「インサイト起点のベネフィット定義」「TTTM フレーム」。再導入しない。
