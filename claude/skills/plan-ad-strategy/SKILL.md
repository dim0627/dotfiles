# plan-ad-strategy（広告戦略立案）

広告クリエイティブの**戦略**を立てる。「どのインサイトに、どの訴求軸で、どんなメッセージを、どの表現フォーマットで当て、何を1変数として検証するか」を出力する。

出力は**それ自体で完結する戦略**であり、誰が受け取って制作してもよい（人間のデザイナー、画像生成、カンプ作成ツール、いずれも一級の消費者）。このスキルは「良い戦略を出す」までが仕事で、**それを実際にどう制作するかには踏み込まない**。制作スペック（アスペクト比・背景の絵柄・フォントサイズ等）は出力に含めない。

このスキルが固定するのは**戦略立案のプロセスとプロの思考フレーム**（普遍・化石化しにくい）。変動する勝ち軸・出稿結果・コンセプトは**外部入力**（product.md / Notion レジストリ）として扱う。これにより出稿0回でも動き、データが溜まるほど精度が上がる。

## 入出力

- **入力**:
  - 対象プロダクトの `docs/product.md`（コンセプト/ペルソナ/コア体験/ポジショニング/トーン&ボイス/ブランド資産）
  - 過去出稿の実績（Notion 広告レジストリ。出稿0回ならスキップ可）
  - プロのマーケ/デザイン原則（このスキルに蒸留済み・下記「思考フレーム」）
- **出力**: 次に撃つクリエイティブの**戦略**（インサイト / 訴求軸 / メッセージ方向 / 表現フォーマット / 検証仮説）。下記「出力スキーマ」の形で提示する。消費者を選ばない粒度（具体コピーは plain text、制作スペックは含めない）

## 前提（満たさなければ止まる）

- **対象プロダクトの `docs/product.md` が存在すること**。コンセプト/ペルソナ/ポジショニング/トーン&ボイスをここから読む。**無ければ「先に docs/product.md を整備してください」と伝えて停止する**（ペルソナや訴求を憶測で捏造しない）
- ペルソナが `docs/product.md` で `[要確認]` 等の未確定なら、戦略の精度が落ちる旨を明示し、確定をユーザーに促してから進む

## プロセス（戦略立案の手順・不変原則）

表現の器から入らない。**Who → What → How の順**で上流から決める。

### STEP 1. インサイトを掘る（Who を深める）
ペルソナの**インサイト**（まだ言語化されていない本音）を、ニーズの3層で掘り下げる:

1. **ニーズ**（自覚された必要性）
2. **潜在ニーズ**（未言語化の欲求・不満）
3. **インサイト**（既成概念に隠れた、行動の本当の動機）

掘る道具に **JTBD（ジョブ理論）** を使う:
- ジョブを **状況(Context) × 動機(Motivation) × 成果(Outcome)** で書く
- ジョブステートメント: **「[状況]のとき、[動機]したい。なぜなら[成果]を得たいから」**
- 乗り換えは **進歩の4つの力**で起きる: **Push(現状の不満) + Pull(新解決策の魅力) > Anxiety(変化への不安) + Habit(惰性)**。優れた訴求は Pull を増やすだけでなく **Anxiety を下げる**点も狙う

### STEP 2. 訴求軸を複数出す（What を広げる）
訴求軸 = **広告で押す戦略的な切り口**（≠ 具体コピーそのもの）。1つのニーズから**複数の訴求軸**を体系的に出す。

- **機能 / 感情 / 社会の3層**で軸を出す（機能だけに偏ると感情・社会層を取りこぼす）。例: 睡眠アプリ → 快適な睡眠(機能) / 朝スッキリ前向き(感情) / 自己管理できている自分(社会)
- 各軸は **ベネフィット訴求**で書く: 機能そのものでなく「その機能が**何を解決するか**」。機能列挙（フィーチャーセリング）にしない
- 軸の発散に詰まったら **消費者ベネフィット6質問**: ①どんな気持ちになりたいか ②誰に認められたいか ③どんな人物でありたいか ④何に楽しみ/不安を感じるか ⑤どんな快楽/苦痛があるか ⑥この機能は競合とどう差別化されるか

> 3層はあくまで**発散の思考フレーム**であり測定可能な法則ではない。軸を広く出す道具として使い、絞り込みは下記「判断ポイント」で人に返す。

### STEP 3. メッセージに変換する（What を言葉に）
各訴求軸を、コピーの主役メッセージに落とす。順序は **ニーズ → 価値 → 訴求軸 → コピー**。

- 機能への「直接的な答え」ではなく「**どういう価値で応えるか**」を言語化する
- **「買うときの気持ち」を起点**に現状へさかのぼって設計する（機能属性からでなく、感情・社会的承認・自己同一性の動機から組む）
- 製品価値は**活用の文脈（状況・目的）**で決まる。機能列挙でなく利用文脈に紐づける

### STEP 4. 表現フォーマットを選ぶ（メッセージが要求する器）
訴求軸ごとに、メッセージ構造が要求する**広告の表現の器**を選ぶ。これは戦略的判断（どの器ならメッセージが立つか）であって、制作スペックではない。

- 器の例: **単発静止画 / カルーセル / 動画 / テキスト主体の投稿**
- 選ぶ基準は**メッセージの構造**: 一撃で刺す単一メッセージ → 単発静止画。ビフォー→アフターや段階的な説明が肝 → カルーセル/動画。会話的・口コミ的に読ませたい → テキスト主体
- **理由を必ず添える**（「なぜこの器か」をメッセージ起点で1文）
- **制作スペックは出さない**: アスペクト比・背景の絵柄・配色・フォントサイズ等は下流（デザイナー/制作スキル）の関心。ここでは決めない
- 利用できる器の実際のメニューは配信プラットフォーム側の仕様に依存する（基本セットは安定しているが、最新メニューは制作時に確認する）

### STEP 5. 検証仮説を立てる（1変数）
**1クリエイティブ = 1訴求軸 / A/B は1変数限定**。複数同時変更は因果が同定できない。

検証対象は**大要素 → 小要素**の順で、出稿ステージに応じて変える:

1. **初期**（勝ち筋未確定）: 大要素 = **訴求軸 / ビジュアル素材 / トンマナ**を検証
2. **中期**（勝ち軸が見えた）: 確立した訴求軸の中で**コピーを精緻化**
3. **成熟期**: **デザイン / 表現フォーマット**を最適化し、クリエイティブ疲弊を防ぎリーチを広げる

各仮説は「**何を変えて、何が上がると見るか**」を1文で書く（例: 「訴求軸を『入力のゆるさ』→『罪悪感ゼロ』に変えるとCTRが上がる、なぜなら主ペルソナの離脱要因が罪悪感だから」）。

訴求軸ごとに以下を提示する。具体コピーは plain text（制作側が改行・サイズ・装飾を決める）。

  "strategy_axes": [

## mandatories（戦略段階で必ず効かせる）

- **1クリエイティブ = 1訴求軸**（混ぜない）。A/B は1変数限定
- **競合貶め表現 NG**（例「あすけんに挫折した人へ」）。競合は補完関係として扱う
- **AI 生成写真は避ける**（日本市場は AI 生成感への拒否反応が強い）。イラスト/グラフィック系を選ぶ（最終判断は product.md のトーンに従う）
- 表現の器から入らない。**Who → What → How** の順を守る

## 判断ポイント（人に握りを返す）

このスキルは戦略を**自動確定しない**。以下は機械的に決めず、案を複数出してユーザーに選んでもらう。今回の知見調査でも型が確立していない空白でもある:

- **3層（機能/感情/社会）のどの軸から攻めるか**の最終選択（1クリエイティブ=1軸に絞る判断）
- **低トラフィック環境で統計的有意に達しないとき**の暫定判断（最低出稿額・必要配信量・有意未満での打ち切り基準）
- **Anxiety/Habit を下げる具体表現**（社会的証明・返金保証・移行コスト訴求等）の採否

## 思考フレーム リファレンス（蒸留元）

deep-research（2026-06-09・敵対的検証済み）でプロの戦略立案知見を調査し、化石化しにくい普遍プロセスのみ採用した。

- **訴求軸4段階フロー**: 3C分析 → ニーズ言語化 → （統計があれば）優先度付け → 自社の独自の強みと掛け合わせ
- **JTBD（ジョブ理論）**: ジョブ = 状況×動機×成果。機能/感情/社会の3層。進歩の4つの力（Push+Pull > Anxiety+Habit）。出典系統: Christensen / Ulwick(Strategyn) / Moesta
- **ベネフィット訴求**: 機能でなく便益（Features tell, benefits sell）。消費者ベネフィット6質問（FICC）
- **インサイト3層**: ニーズ → 潜在ニーズ → インサイト
- **A/Bテスト設計**: 1変数原則 / 大要素→小要素 / ステージ別検証

> 検証で**反証され不採用**にしたもの: 「訴求軸の8パターン固定分類」「インサイト起点のベネフィット定義」「TTTM フレーム」。再導入しない。
