# plan-ad-strategy（広告戦略立案）

X(Twitter)広告クリエイティブの**戦略**を立てる。「どのインサイトに、どの訴求軸で、どんなメッセージを、どんなクリエイティブ方向で、どの器（表現モード）で当て、各スロットのコピーをどう振り、何を1変数として検証するか」を**複数候補**で出力する。

このスキルは**良い戦略候補を複数出すまで**が仕事。1つに絞らない（絞り込みは人の握り）。出力は**それ自体で完結する戦略まとめ**で、誰が受け取って制作してもよい（人間のデザイナー、画像生成、カンプ作成、いずれも一級の消費者）。**実際にどう制作するか（数値スペック・文字組み・絵そのもの）には踏み込まない**。

## アウトプットの流れ

```
docs/product.md（＋あれば運用レポート・過去出稿）
      │
      ▼
[plan-ad-strategy] ── 戦略候補を複数出す（Markdown）
      │
      ▼
  人が握る ── どの戦略を今回撃つか選ぶ（自動確定しない）
      │
      ▼
   制作 ── 選ばれた戦略が要求するクリエイティブを作る
```

## 管轄マップ（どこまでがこのスキルか）

| レイヤー | 中身 | 管轄 |
|---|---|---|
| **戦略** | インサイト / 訴求軸 / メッセージ方向 / **クリエイティブ方向（ラフ）** / **器の選択（表現モード）** / **全スロットのコピー方向** / 検証仮説 | **このスキル** |
| **制作** | 正確なパレット(hex)・フォントサイズ(px)・比率・構図座標・実コピーの文字組み・背景素材の生成・PNG 書き出し | 下流（制作） |
| **配信** | 入札・ターゲティング・予算・最新の広告メニュー確認 | 配信 / 人 |

> 「その戦略に必要なクリエイティブ」を2つに割る。**クリエイティブ・ブリーフ（＝方向：クリエイティブ方向＋器＋全スロットのコピー方向＋理由）はこのスキルの出力に入る**。**クリエイティブ実物（＝数値スペック・文字組み・絵そのもの）は下流の制作**。この線引きを越えない。
>
> 恒久的なブランドDNA（パレット・書体・形・イラスト基調・トーン&ボイス）の**正本は `docs/product.md`**。このスキルはそれを各訴求軸に**適用したラフな方向**を言葉で出すだけで、パレットや px を再定義しない。

## 前提（満たさなければ止まる）

- **対象プロダクトの `docs/product.md` が存在すること**。コンセプト/ペルソナ/ポジショニング/トーン&ボイス/ブランド資産をここから読む。**無ければ「先に docs/product.md を整備してください」と伝えて停止する**（ペルソナや訴求を憶測で捏造しない）。

## 起動時の確認（入力を決め打ちしない）

product.md を読んだら、**そのまま戦略生成に進まず、まず「広告運用レポート（X Ads Manager 等の実績）や過去出稿の記録はあるか」をユーザーに一度問いかける**:

- **ある** → 受け取って検証仮説・次の打ち手に反映する（実績ベースで精度が上がる）。
- **ない** → その旨を明示して進む（product.md だけでも戦略候補は出せる。ただし検証仮説は実績でなく当て込みになる点を添える）。

## 入力

- **`docs/product.md`（正本・必須）**: プロダクト知識の正本。具体的に何を読むかは下記「product.md に期待する情報」。
- **広告運用レポート（あれば）**: X Ads Manager 等の実績（CTR / CVR・勝ち負けクリエイティブ）。あると検証仮説と次の打ち手の精度が上がる。**無くても動く**（出稿0回でも戦略は出せる）。
- **過去出稿の記録（あれば）**: Notion 広告レジストリ等のトランザクショナルな記録。無ければスキップ。
- **プロのマーケ/デザイン原則**: このスキルに蒸留済み（下記「進め方」のフレーム）。

## product.md に期待する情報（このスキルの入力契約）

このスキルは下記を `docs/product.md` から読む。**product.md が無い／薄いときは、この一覧をもとに整備を手伝える**（「このスキルを使いたいので product.md を作って」から始められる）。各項目は "何のために要るか" とセットで持つ:

- **コンセプト / コア体験**: 何のプロダクトで、ユーザーが得る中心価値は何か。→ ベネフィット訴求の素材。
- **ペルソナ**: 誰か。状況・動機・不満・離脱要因まで。→ Who（インサイト・JTBD）の土台。**ここが薄いと戦略全体の精度が落ちる**。
- **ポジショニング**: 競合との関係（差別化・補完）。→ What（差別化ベネフィット）と、競合の扱い方針。
- **トーン&ボイス**: 語り口・人格。→ メッセージ方向とコピーのトーン。
- **ブランドDNA / アートディレクション**: パレット・書体・形・イラスト or 写真の方針・do/don't。→ **クリエイティブ方向の正本**（このスキルはこれを各訴求軸に適用するだけで、再定義しない）。
- **ブランド方針（禁則）**: 写真の可否・競合の扱い等。→ mandatories が参照する。

> このスキルが消費するのは上記まで。配信・制作だけが要る項目（入札設定・正確な px 等）は product.md の他用途に委ね、ここでは要求しない。

## 出力

次に撃つクリエイティブの**戦略候補（複数）**を、人が読んで選べる **Markdown の戦略まとめ**で提示する（下記「出力フォーマット」）。クリエイティブ方向・コピーはすべて plain text の「方向」まで。数値スペック（hex・px・比率・座標）は含めない。

## X広告のアナトミー（コピーを振る先のスロット）

戦略は「全スロットのコピー方向」を出す。**どのスロットが存在するかは選んだフォーマットで変わる**:

- **常にある**:
  - **投稿本文**: ツイート文（メディアの上）。
  - **メディア内コピー**: 画像/動画に乗る文字（小ラベル / 見出し / サブ）。カルーセルは**2〜6枚**スワイプで、各カードに個別のコピー＋遷移先を持てる。
- **カード/ボタン型フォーマット（Website Card 等）のときだけ**:
  - **カード見出し**（目安70字以内で安定表示）／ **CTA ボタン**（「詳しくはこちら」等）。

選んだ器に存在するスロットだけコピー方向を振る（存在しないスロットは出さない）。

> セーフゾーン注意: 画像の**左下**に X 側のキャプション/カードUIが重なる。重要要素を左下隅に置かないことは方向として意識する（具体の配置は制作の管轄）。

## 進め方（Who → What → How の順序原則）

**番号を機械的に踏むパイプラインではなく、上流から下流へ決める順序原則**。表現の器から入らず、Who（誰の何のジョブか）→ What（何を訴求し何と言うか）→ How（どんな見え方で、どの器で、どのスロットに、どう検証するか）の順で固める。

### Who — インサイトを掘る
ペルソナの**インサイト**（まだ言語化されていない本音）をニーズの3層で掘る:

1. **ニーズ**（自覚された必要性）
2. **潜在ニーズ**（未言語化の欲求・不満）
3. **インサイト**（既成概念に隠れた、行動の本当の動機）

掘る道具に **JTBD（ジョブ理論）**:
- ジョブを **状況(Context) × 動機(Motivation) × 成果(Outcome)** で書く。
- ジョブステートメント: **「[状況]のとき、[動機]したい。なぜなら[成果]を得たいから」**。
- 乗り換えは **進歩の4つの力**で起きる: **Push(現状の不満) + Pull(新解決策の魅力) > Anxiety(変化への不安) + Habit(惰性)**。優れた訴求は Pull を増やすだけでなく **Anxiety を下げる**点も狙う。

### What — 訴求軸を複数出し、メッセージに変換する
訴求軸 = **広告で押す戦略的な切り口**（≠ 具体コピーそのもの）。1つのニーズから**複数の訴求軸**を体系的に出す:

- **機能 / 感情 / 社会の3層**で軸を出す（機能だけに偏ると感情・社会層を取りこぼす）。例: 睡眠アプリ → 快適な睡眠(機能) / 朝スッキリ前向き(感情) / 自己管理できている自分(社会)。
- 各軸は **ベネフィット訴求**で書く: 機能そのものでなく「その機能が**何を解決するか**」。機能列挙（フィーチャーセリング）にしない。
- 発散に詰まったら **消費者ベネフィット6質問**: ①どんな気持ちになりたいか ②誰に認められたいか ③どんな人物でありたいか ④何に楽しみ/不安を感じるか ⑤どんな快楽/苦痛があるか ⑥この機能は競合とどう差別化されるか。

各訴求軸を主役メッセージに変換する。順序は **ニーズ → 価値 → 訴求軸 → コピー**:

- 機能への「直接的な答え」でなく「**どういう価値で応えるか**」を言語化する。
- **「買うときの気持ち」を起点**に現状へさかのぼって設計する（機能属性からでなく、感情・社会的承認・自己同一性の動機から組む）。
- 製品価値は**活用の文脈（状況・目的）**で決まる。機能列挙でなく利用文脈に紐づける。

> 3層はあくまで**発散の思考フレーム**であり測定可能な法則ではない。軸を広く出す道具として使い、絞り込みは下記「判断ポイント」で人に返す。

### How — クリエイティブ方向・器・スロット別コピー・検証仮説を決める

**1. クリエイティブ方向（ラフ）**: ブランドDNA（`docs/product.md`）をこの訴求軸のメッセージに適用した、**ラフなビジュアルの方向**を言葉で出す:

- **狙うムード**（例: 温かく肯定的 / 緊張感のあるビフォーアフター）
- **主役と構図の方向**（何を画の主役にするか・コピー用の余白をどこに取るか）
- **やる・やらない（do / don't）**（例: 説教感のある人物は出さない）

**数値・hex・px・座標・正確なパレットは出さない**（それは制作）。パレットや書体は product.md が正本なので、ここでは「product.md のブランドDNAに沿って」適用方向を示すだけ。

**2. 器（表現モード）**: クリエイティブ方向とメッセージ構造から器が落ちてくる。器ありきで選ばない:

- **単発静止画**: 一撃で刺す単一メッセージ。段階的な見せ方が不要。
- **カルーセル（2〜6枚）**: ビフォー→アフター、段階的な説明、複数便益の列挙が肝。**何枚で何を見せるか**まで方向づける。
- **動画**: 時間軸・動き・実演が便益の核。
- **テキスト主体**: 会話的・口コミ的に読ませたい。

器は**名前まで決める**（コピー構造が器に依存するため。例: カルーセルなら何パネル分のコピーが要るかが変わる）。**理由を1文添える**。制作スペックには踏み込まない。

**3. スロット別コピー方向**: 選んだ器に存在するスロットへコピーの方向を振る（上記「アナトミー」参照）。各スロットは plain text の「方向」で書き、文字組み・装飾・サイズは制作に委ねる。

**4. 検証仮説（1変数）**: **1クリエイティブ = 1訴求軸 / A/B は1変数限定**（複数同時変更は因果が同定できない）。**運用レポート（入力）があればその実績を踏まえて**次に動かす1変数を選ぶ。各仮説は「**何を変えて、何が上がると見るか＋なぜ**」を1文で書く。

## 出力フォーマット（Markdown・複数候補）

戦略候補を**複数**、人が比較して選べる体裁で並べる。各候補は次の構造（※下記は構造を示すダミー。実値は product.md から）:

```markdown
## キャンペーンゴール
無料トライアル登録（product.md ／ ユーザー確認）

## ペルソナ・インサイト
（product.md のペルソナから。例: 始めたいが「続かない自分」を直視したくなくて踏み出せない）

---

## 候補A：〈訴求軸名〉
- **訴求軸**: 〈切り口〉（層: 感情）
- **JTBD**: [状況]のとき[動機]したい。なぜなら[成果]を得たいから
- **ベネフィット**: 機能でなく便益で1行
- **メッセージ方向**: 何を伝えるか
- **クリエイティブ方向**: 狙うムード ／ 主役と構図の方向 ／ do・don't ／ コピー用余白（※言葉のみ・数値なし）
- **表現モード**: 単発静止画 ／ 理由: 一撃で刺す単一メッセージで段階的な見せ方は不要だから
- **スロット別コピー方向**:
  - 投稿本文: 〈方向〉
  - メディア内: 〈主役メッセージの方向〉
  - （カード型なら）カード見出し: 〈方向〉（目安70字以内）
  - （カード型なら）CTA: 〈アクションの方向〉
- **検証仮説**: 何を変えると何が上がる＋なぜ
- **今回動かす変数**: 訴求軸

---

## 候補B：〈別の層の訴求軸〉
（同じ構造。機能/感情/社会を取りこぼさないよう層をまたぐ）

## 判断ポイント（あなたが握る）
- どの候補を今回撃つか（下記「判断ポイント」参照）
```

- 候補は**機能/感情/社会の3層を取りこぼさない**ように、層をまたいで複数出す。
- カルーセルを選んだ候補は、メディア内コピーを**パネル順の方向**で書く。
- クリエイティブ方向・コピーはすべて plain text の「方向」。**数値・hex・px・比率は持たせない**。

## mandatories（戦略段階で必ず効かせる）

- **1クリエイティブ = 1訴求軸**（混ぜない）。A/B は1変数限定。
- 表現の器から入らない。**Who → What → How** の順を守る。
- **クリエイティブ方向・コピーは言葉の方向まで**。**数値・hex・px・座標・正確なパレット・実コピーの文字組みは出さない**（制作の管轄）。
- **恒久的なブランド方針（写真の可否・競合の扱い・トーン等）は `docs/product.md` に従う**。product.md に無い方針を憶測で足さない。

## 判断ポイント（人に握りを返す）

このスキルは戦略を**自動確定しない**。以下は機械的に決めず、候補を複数出してユーザーに選んでもらう:

- **どの候補を今回撃つか**＝3層（機能/感情/社会）のどの軸から攻めるかの最終選択（1クリエイティブ=1軸に絞る判断）。
- **低トラフィックで統計的有意に達しないとき**の暫定判断（必要配信量・打ち切り基準）。
- **Anxiety/Habit を下げる具体表現**（社会的証明・返金保証・移行コスト訴求等）の採否。
